CS250 Value Noise

WebGL2 / C++ / Emscripten

Project Overview

Value Noise is a CS250 Computer Graphics assignment that implements procedural noise generation entirely on the CPU using C++. A permutation hash table maps lattice coordinates to random values, and smooth interpolation (Quintic, Smoothstep, Cosine, or Linear) blends adjacent values to produce continuous noise. The generated texture is uploaded to the GPU each frame and rendered via WebGL2.

The demo supports 1D, 2D, and 3D noise, multiple pattern modes (Plain Value, Fractal Sum, Turbulence, Marble, Wood), and interactive controls for period dimension, texture resolution, tiling scale, and value sets — all running in real time using a multi-threaded job system on native builds and single-threaded incremental generation on the web.

C++ WebGL2 GLSL ES 3.0 Emscripten OpenGL ES 3.0 ImGui CMake

Tasks Completed

Permutation Hash Table

  • Built a shuffle-based permutation table for period-repeating hash lookups
  • Supports 1D, 2D, and 3D coordinate hashing with configurable period (2–16384)
  • Guarantees tiling — noise wraps seamlessly at any period boundary

Smooth Interpolation

  • Implemented 4 fade functions: Linear, Cosine, Smoothstep, Quintic
  • Applied bilinear and trilinear mixing for 2D/3D noise
  • Quintic (6t⁵ − 15t⁴ + 10t³) produces zero first and second derivatives at lattice points

Fractal Patterns

  • Fractal Sum: octave stacking with lacunarity=2, gain=0.5
  • Turbulence: absolute value of each octave for sharp ridges
  • Marble & Wood: sin-wave distortion driven by fractal noise

CPU Texture Generation

  • Generates RGBA textures up to 4096×4096 on the CPU
  • Native: parallel generation via multi-threaded job system
  • Web: incremental row-by-row generation capped at ~31 ms/frame to stay responsive

Challenge & Lesson

Challenge — Texture Filtering Bug

With period = texture size (1 pixel per lattice cell), the output should show sharp solid-color blocks. Instead it appeared blurry — the GPU was magnifying pixels with GL_LINEAR. The root cause was that LoadFromMemory hardcoded GL_LINEAR without syncing the member variable, so a subsequent SetFiltering(NearestPixel) call silently became a no-op due to an early-return guard. Fixing the sync made nearest-pixel rendering work correctly.

Challenge — Frequency & the Mosaic Effect

Value noise looks smooth when frequency = period / textureSize is small (many pixels per cell). When frequency = 1.0 every pixel lands exactly on a lattice point, the interpolant is always 0, and the result is solid-color blocks — the classic "mosaic" look. Understanding this ratio was key to controlling the visual output.

Lesson — CPU vs GPU Noise

Generating noise on the CPU and uploading a texture each frame is simple to debug but expensive at high resolutions. Implementing the same math in a fragment shader would evaluate noise per-pixel in parallel on the GPU — no texture upload needed — and scale to any resolution for free.

Lesson — Tiling Through Permutation

Using x & mask instead of x % period in the hash lookup makes tiling essentially free and branch-free, as long as the period is a power of two. This is why all noise libraries restrict period sizes to powers of two.
