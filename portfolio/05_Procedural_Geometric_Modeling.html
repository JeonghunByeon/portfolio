CS250 Procedural Geometric Modeling

WebGL / GLSL Graphics

Project Overview

Procedural Geometric Modeling is a CS250 Computer Graphics assignment focused on algorithmically generating 3D mesh geometry at runtime. Rather than loading pre-made models, the program constructs a Sphere, Cylinder, and Torus from scratch using parametric equations — computing vertex positions, normals, and index buffers entirely in code. The meshes are rendered with WebGL and support interactive adjustment of slices and stacks to observe how subdivision density affects surface smoothness.

Tasks Completed

Mesh Generation

  • Implemented Sphere generation using latitude/longitude parametric equations with configurable slices & stacks
  • Built Cylinder mesh with capped top/bottom faces and adjustable radial segments
  • Constructed Torus geometry by revolving a circle profile around a central axis

Index Buffer & Normals

  • Built an IBO (Index Buffer Object) to share vertices across triangles, minimizing GPU memory usage
  • Computed per-vertex surface normals analytically from parametric derivatives
  • Mapped UV texture coordinates across each mesh surface

Interactive Controls

  • Exposed slices & stacks parameters to the UI for real-time mesh re-generation
  • Added mesh switching to toggle between Sphere, Cylinder, and Torus at runtime
  • Wired up a wireframe toggle to visualize the underlying triangle topology

Challenge & Lesson

Challenge — Torus Index Winding

Generating correct triangle indices for a torus was error-prone because both the tube and the ring wrap around — the last column of quads must stitch back to the first. Off-by-one errors in the index loop produced visible seams or degenerate triangles that only appeared at certain slice counts.

Challenge — Smooth Normals at Poles

At the top and bottom poles of a sphere, all vertices in a stack ring share the same position but need different normals for correct shading. Handling this degenerate case required special-casing the pole vertices rather than using the general parametric formula.

Lesson — Parametric Thinking

Expressing every shape as a function of two parameters (u, v) is a powerful abstraction. Once I had a working sphere generator, the cylinder and torus followed naturally by just changing the equations — the buffer-building code remained identical.

Lesson — Index Buffers Pay Off

Switching from drawArrays to drawElements with an IBO cut vertex data size by roughly 4× on a high-subdivision sphere. Seeing that number drop in the profiler made the extra indexing complexity feel immediately worthwhile.
