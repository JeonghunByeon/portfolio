CS250 Hello Quad

WebGL / GLSL Graphics

Project Overview

Hello Quad is the first WebGL assignment in DigiPen's CS250 Computer Graphics course. Starting from scratch, the task involves initializing a WebGL context, writing GLSL vertex and fragment shaders by hand, and rendering a colored quad to the screen — covering the full graphics pipeline end to end. The goal is to build a concrete understanding of buffer creation, shader compilation, and the draw-call flow that underpins all real-time GPU rendering.

Tasks Completed

WebGL Pipeline Setup

  • Acquired a WebGL rendering context from the canvas element
  • Configured the viewport and set the background clear color
  • Enabled GL extensions and built an error-handling routine

GLSL Shader Authoring

  • Vertex Shader: transforms vertex positions into clip space
  • Fragment Shader: outputs a uniform color per pixel
  • Compiled and linked shaders into a WebGL program object

Vertex Buffer Construction

  • Defined 6 vertices (2 triangles) composing a quad
  • Created a VBO and uploaded vertex data to the GPU
  • Issued the final draw call via gl.drawArrays(TRIANGLES)

Challenge & Lesson

Challenge — Shader Compile Debugging

GLSL compile errors only surface at runtime, so a single typo produces a completely black screen with no obvious feedback. I built an error-logging routine around gl.getShaderInfoLog() that prints the exact error line, which cut shader debugging time dramatically.

Challenge — Understanding the Coordinate System

WebGL's clip space has Y pointing up and a range of −1 to +1, which clashes with the pixel-space intuition most developers start with. Manually placing vertices and observing the results was the fastest way to internalize NDC (Normalized Device Coordinates).

Lesson — What APIs Actually Abstract

Manually wiring CPU data into a VBO, uploading it to the GPU, and watching shaders consume it made it crystal clear what Unity and high-level OpenGL wrappers are hiding. Every abstraction became easier to reason about after seeing the raw layer underneath.

Lesson — Thinking in Pipelines

Rendering is not "drawing a picture" — it's data flowing through a fixed-function pipeline. That mental shift made every subsequent assignment (transforms, lighting, textures) faster to pick up, because the structure was already familiar.
