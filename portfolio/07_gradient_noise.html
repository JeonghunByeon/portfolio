CS250 Gradient Noise

WebGL2 / GLSL / C++ / Emscripten

Project Overview

Gradient Noise (Perlin Noise) is a CS250 Computer Graphics assignment that implements Ken Perlin's Improved Gradient Noise algorithm entirely on the GPU using GLSL. Unlike the previous Value Noise assignment — which computed noise on the CPU and uploaded a texture each frame — all noise evaluation happens in a fragment shader, writing into an offscreen R32F framebuffer at full floating-point precision.

A hardcoded period-64 permutation table and 12 fixed gradient directions produce seamlessly tiling 3D noise. A second pass renders the stored noise texture as either a flat 2D slice or a displaced 3D terrain surface, with ImGui controls for pattern type, tiling scale, Z-depth animation, and surface geometry parameters.

C++ GLSL ES 3.0 WebGL2 OpenGL 4.2+ Emscripten ImGui CMake

Tasks Completed

GPU Gradient Noise in GLSL

  • Implemented Improved 3D Perlin Noise in a fragment shader
  • Hardcoded period-64 permutation table (doubled to 128 entries) to avoid bounds issues
  • Used 12 fixed gradient directions cycled via hash & 15 for consistent distribution
  • Quintic fade function 6t⁵ − 15t⁴ + 10t³ ensures zero first and second derivatives at lattice boundaries

Five Noise Patterns

  • Plain Gradient — raw noise remapped from [−1, 1] to [0, 1]
  • Fractal Sum (fBm) — 5 octaves with lacunarity 2, gain 0.5
  • Turbulence — absolute-value fBm for sharp ridge features
  • Marble — sine bands distorted by turbulence (integer band count ensures tiling)
  • Wood — concentric rings via fract(noise × 10)

R32F Framebuffer & Texture System

  • Extended Texture class with ColorFormat enum: RGBA8, RGBA32F, R32F
  • New LoadAsFormat() uses TexStorage2D on GL 4.2+ / WebGL and falls back to TexImage2D on older drivers
  • Updated FrameBuffer::ColorComponent enum and LoadWithSpecification() to support all three formats
  • Noise values stored as single-channel 32-bit float — no precision loss from RGBA quantization

Dual Visualization Modes

  • Texture mode — full-screen quad samples the R32F noise texture as grayscale
  • Surface mode — a 200×200 plane mesh displaces vertices along local Z using noise height values, forming 3D terrain
  • Spectator camera (WASD + mouse drag) for exploring the surface in real time
  • ImGui sliders for area scale, height scale, stack/slice density, tiling scale, and Z animation depth

Challenge & Reflection

Challenge — Seamless Tiling

Making the noise tile seamlessly at any display scale was the trickiest constraint. The key insight: noise is periodic with period equal to the permutation table size (64). By mapping texture coordinates [0, 1] to noise input [0, 64], the noise completes exactly one period across the texture — so it tiles perfectly at any integer or fractional display scale without any special border handling.

Challenge — R32F Framebuffer on WebGL

Supporting both desktop OpenGL and WebGL required careful branching in LoadAsFormat(). TexStorage2D is available on GL 4.2+ and WebGL2, but older desktop drivers require TexImage2D with the correct pixel format / type pair for each internal format (e.g. GL_RED / GL_FLOAT for R32F). Getting these combinations wrong produced silent black output with no GL error.

Lesson — GPU vs CPU Noise

Moving from CPU-side noise (Value Noise assignment) to GPU-side evaluation was a significant leap in scalability. The fragment shader evaluates noise independently per pixel in parallel, so a 1024×1024 noise texture generates in a single draw call regardless of resolution. There is no texture upload cost, and the Z slider animates through 3D noise slices with zero latency.

Lesson — Gradient vs Value Noise

Value noise assigns random scalars to lattice points and interpolates between them, which tends to produce a blocky, axis-aligned look. Gradient noise assigns random direction vectors instead and computes dot products with offset vectors — producing smoother, more natural-looking results. The Improved Perlin version further restricts gradients to 12 fixed directions, eliminating axis-aligned bias and improving visual quality.
